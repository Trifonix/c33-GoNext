# PLAN.md  
## План реализации GoNext — «Дневник туриста»

Документ описывает поэтапный порядок действий по реализации MVP из [PROJECT.md](PROJECT.md).  
Стек уже задан: React Native, Expo, Expo Router, React Native Paper, SQLite, офлайн.

---

## Этап 0. Подготовка и каркас ✅

**Статус:** выполнен.

1. Проверить текущий каркас (Expo Router + Paper + TypeScript) и зависимости.
2. Добавить недостающие пакеты: SQLite (`expo-sqlite`), файловая система (`expo-file-system`), выбор фото (`expo-image-picker`), открытие карт/навигатора (`expo-linking` / Linking), при необходимости даты (`@react-native-community/datetimepicker`).
3. Зафиксировать структуру папок проекта, например:
   - `app/` — экраны (Expo Router)
   - `src/db/` — схема и доступ к SQLite
   - `src/types/` — типы Place, Trip, TripPlace
   - `src/services/` — бизнес-логика (следующее место, карты, фото)
   - `src/components/` — переиспользуемые UI-компоненты на Paper
4. Настроить корневой `_layout.tsx`: PaperProvider, тема, базовая навигация.

**Критерий готовности:** приложение запускается; структура папок и типы сущностей согласованы с PROJECT.md.

## Этап 1. Модель данных и локальное хранилище ✅

**Статус:** выполнен.

1. Описать TypeScript-типы:
   - `Place` (id, name, description, visitlater, liked, dd, photos[], createdAt)
   - `Trip` (id, title, description, startDate, endDate, places[], createdAt, current)
   - `TripPlace` (placeId, order, visited, visitDate, notes, photos[])
2. Спроектировать схему SQLite с связью Trip → TripPlace → Place.
3. Реализовать инициализацию БД при старте приложения (создание таблиц, миграции при необходимости).
4. Реализовать CRUD-репозитории:
   - места: создать / список / получить / обновить / удалить
   - поездки: создать / список / получить / обновить / удалить / пометить `current`
   - места в поездке: добавить / изменить порядок / отметить посещённым / заметки и фото
5. Определить хранение фото: локальные пути в файловой системе приложения + ссылки в БД.

**Критерий готовности:** данные сохраняются локально и читаются после перезапуска без интернета.

## Этап 2. Главный экран и навигация ✅

**Статус:** выполнен.

1. Переделать Home в главный экран GoNext:
   - название программы
   - вертикальный ряд кнопок Paper: «Места», «Поездки», «Следующее место», «Настройки»
2. Настроить маршруты Expo Router для экранов:
   - список мест / карточка места / форма места
   - список поездок / детали поездки / форма поездки
   - «Следующее место»
   - настройки (заглушка или минимальный экран)
3. Обеспечить простую мобильную навигацию без сложных вложенных меню; UI — компоненты React Native Paper.
4. Добавить единое фоновое изображение `assets/backgrounds/gonext-bg.png` через
   переиспользуемый компонент на базе `ImageBackground`.

**Критерий готовности:** из главного экрана открываются все основные разделы, работает
возврат назад, а фон заполняет область приложения на всех экранах.

## Этап 3. Режим «Места» ✅

**Статус:** выполнен.

1. Экран списка мест (FlatList + Paper Card/List).
2. Экран создания/редактирования места: название, описание, флаги `visitlater` / `liked`, координаты `dd`.
3. Прикрепление фотографий (выбор из галереи / камеры → сохранение в локальное хранилище).
4. Детальная карточка места: просмотр полей и фото.
5. Действие «Открыть на карте» по координатам.
6. Возможность использовать место позже при создании поездки (выбор из базы мест).

**Критерий готовности:** полный цикл работы с местами офлайн; место открывается на карте при наличии координат.

---

## Этап 4. Режим «Поездки» ✅

**Статус:** выполнен.

1. Экран списка поездок с признаком текущей (`current`).
2. Создание поездки: название, описание, даты начала/конца.
3. Добавление мест в поездку: из базы мест или создание нового места.
4. Упорядочивание маршрута (`order`: вверх/вниз или drag-and-drop, если уместно в MVP).
5. Детали поездки: просмотр маршрута как плана.
6. Отметка места как посещённого (`visited`, `visitDate`).
7. Заметки и фото к месту в рамках поездки (TripPlace).
8. Переключение «текущая поездка» (только одна `current = true`).
9. Отображение поездки и как плана, и как дневника (по факту посещений).

**Критерий готовности:** можно спланировать маршрут, отметить посещения и вести дневник по поездке.

## Этап 5. Режим «Следующее место» ✅

**Статус:** выполнен.

1. Сервис: взять активную поездку (`current = true`) → первое место с `visited = false`.
2. Экран «Следующее место»: информация о месте (название, описание, порядок в маршруте).
3. Обработка пустых состояний: нет текущей поездки / все места посещены.
4. Кнопки:
   - открыть на карте
   - открыть в навигаторе (Google Maps / системный навигатор)
5. Опционально для удобства: быстрая отметка «посещено» с экрана следующего места.

**Критерий готовности:** пользователь за один тап видит следующее место и может открыть навигацию.

## Этап 6. Настройки и полировка MVP ✅

**Статус:** выполнен.

1. Экран «Настройки»: минимальный набор (версия приложения, краткая справка; без сервера и аккаунтов).
2. Единообразные пустые состояния, ошибки и Snackbar через Paper.
3. Проверка офлайн-сценариев: CRUD, фото, навигация по сохранённым координатам.
4. Базовая проверка на Android и iOS (Expo Go / эмулятор).
5. Обновить README под реальные возможности MVP (не демо-кнопку).
6. Проверить адаптивность общего фона на Android, iOS и web:
   - фон заполняет экран с `resizeMode="cover"`;
   - корневая web-страница не получает лишнюю прокрутку из-за изображения;
   - длинное содержимое прокручивается внутри `ScrollView` / `FlatList`.

**Критерий готовности:** MVP соответствует разделам 1–6 PROJECT.md; пункты «Будущее развитие» не входят в объём.

## Этап 7. Вне объёма MVP (осознанно отложить) ✅

**Статус:** выполнен — решение зафиксировано, функции **не реализованы** (по плану).

Следующие возможности из раздела 7 [PROJECT.md](PROJECT.md) **намеренно исключены из MVP**:

| Функция | Почему отложено | Возможное направление (позже) |
|---------|-----------------|-------------------------------|
| Экспорт поездки в PDF | Не входит в MVP | `expo-print` / генерация HTML → PDF |
| Резервное копирование | Нужен формат экспорта/импорта | JSON/ZIP с БД и фото через `expo-sharing` |
| Синхронизация между устройствами | Требует сервер или облако | Backend + auth или iCloud/Google Drive |
| Совместные поездки | Мультипользовательская модель | Аккаунты, приглашения, общая БД |
| Статистика путешествий | Аналитика поверх накопленных данных | Агрегации по `trips` / `trip_places` |
| Маршрут целиком на карте | Сложнее одной точки | `react-native-maps` + polyline по координатам |

**Критерий готовности этапа:** границы MVP явно определены; в коде нет заготовок под эти функции, чтобы не размывать scope.

## Этап 8. Светлая / тёмная тема

**Статус:** не начат.

1. В экране «Настройки» добавить выбор темы: светлая / тёмная.
2. Сохранять выбранную тему локально и применять её при следующем запуске.
3. Подключить выбранную тему к `PaperProvider` (и при необходимости к корневому layout).
4. При выборе **тёмной** темы убирать фоновое изображение (не показывать `ImageBackground` / общий фон).
5. При выборе **светлой** темы снова показывать фоновое изображение, как в этапе 2.

**Критерий готовности:** в настройках можно переключить светлую/тёмную тему; в тёмной теме фон отсутствует, в светлой — отображается.

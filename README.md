# GoNext

**GoNext** — мобильное приложение «Дневник туриста» на **Expo**, **React Native**, **Expo Router** и **React Native Paper**.

Приложение работает **полностью офлайн**: места, поездки, маршруты, заметки и фото хранятся локально на устройстве (SQLite + файловая система). Регистрация и сервер не требуются.

## Возможности MVP

### Главный экран
- Быстрый переход к разделам: **Места**, **Поездки**, **Следующее место**, **Настройки**
- Единое фоновое изображение на всех экранах с адаптивным заполнением области приложения

### Места
- Список мест с флагами «хочу посетить» / «понравилось»
- Создание и редактирование: название, описание, координаты (DD), фото
- Просмотр карточки места, открытие на карте
- Места можно добавлять в поездки

### Поездки
- Список поездок с признаком **текущей** поездки
- Создание поездки: название, описание, даты
- Маршрут: добавление мест из базы или создание новых, изменение порядка
- Режимы **План** и **Дневник**
- Отметка посещений, заметки и фото к каждой точке маршрута

### Следующее место
- Следующая непосещённая точка **текущей** поездки
- Открытие на карте и в навигаторе (Google Maps)
- Быстрая отметка «посещено»

### Настройки
- Версия приложения, справка, информация о локальном хранении

## Стек

- [Expo](https://expo.dev/) ~57
- [Expo Router](https://docs.expo.dev/router/introduction/) — файловая навигация

## Структура проекта

```
app/                    # экраны (Expo Router)
  index.tsx             # главный экран
  places/               # режим «Места»
  trips/                # режим «Поездки»
  next.tsx              # «Следующее место»
  settings.tsx          # настройки
src/
  db/                   # SQLite: схема, репозитории
  services/             # карты, фото, следующее место
  components/           # переиспользуемые UI-компоненты
  types/                # типы Place, Trip, TripPlace
  utils/                # координаты, даты
assets/
  backgrounds/         # фон интерфейса (gonext-bg.png)
  ...                  # иконки и splash
```

## Требования

- Node.js 18+
- npm
- [Expo Go](https://expo.dev/go) на телефоне или эмулятор Android / симулятор iOS

## Установка и запуск

```bash
npm install
npm start
```

Затем:

- `a` — Android (Expo Go / эмулятор)
- `i` — iOS (Expo Go / симулятор)
- отсканируйте QR-код в Expo Go

Отдельные команды:

```bash
npm run android
npm run ios
```

> **Примечание:** целевые платформы MVP — **Android и iOS** (Expo Go). Web поддерживается с оговорками: для `expo-sqlite` нужны настройка Metro (`.wasm`) и заголовки COOP/COEP — они уже в `metro.config.js`. После изменений конфига перезапустите `npm start` и откройте `http://localhost:8081` заново.

На всех экранах используется `ImageBackground` с изображением
`assets/backgrounds/gonext-bg.png`. На web корневой контейнер ограничен высотой окна,
чтобы фон располагался за содержимым и не создавал прокрутку страницы. Внутренние
`ScrollView` и `FlatList` продолжают прокручиваться на экранах с длинным содержимым.

## Проверка офлайн-сценариев

Рекомендуемый чек-лист перед релизом MVP:

1. **Места** — создать, отредактировать, прикрепить фото, открыть на карте (при наличии координат)
2. **Поездки** — создать поездку, добавить места, изменить порядок, отметить посещение, добавить заметки/фото
3. **Текущая поездка** — переключить `current`, проверить экран «Следующее место»
4. **Перезапуск** — закрыть и снова открыть приложение: данные и фото должны сохраниться
5. **Офлайн** — включить режим «в самолёте» / отключить сеть: CRUD и просмотр фото работают; карта/навигатор открываются по сохранённым координатам (нужен установленный картографический app)
